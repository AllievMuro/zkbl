How It Works

Four Steps from Event to Verified Receipt

ZKBL intercepts privileged on-chain events and replaces raw calldata with cryptographic compliance receipts — all without changing your existing deploy pipeline.

1

Define Policy in YAML

Specify multisig thresholds, timelock durations, and caller allowlists in a human-readable YAML file alongside your contracts.

2

Event Fires on BSC

BEP-20 mint, ownership transfer, or BEP-177 governance execution triggers ZKBL's Node.js proof server via event hook.

3

Groth16 Proof Generated

The proof server compiles the event against your YAML rules, generating a Groth16 ZK proof via SnarkJS in under 30 seconds.

4

Receipt Posted On-Chain

A verifiable compliance receipt is stored in the public BSC registry contract, bound to the block hash and a per-event nonce.

Features

Built for DeFi Teams Closing Institutional Deals

Core Protocol

YAML-to-Circom Compiler + On-Chain Registry

Write compliance rules in YAML — multisig thresholds, timelock durations, caller allowlists. ZKBL compiles them to Circom 2.x circuits automatically, generates Groth16 proofs via SnarkJS, and posts receipts to a Solidity 0.8.x registry on BSC testnet and mainnet. Receipts are bound to BNB block hashes and per-event nonces for tamper-proof audit trails.

SDK

One-Line Hardhat & Foundry Integration

Add ZKBL to your existing deploy pipeline with a single config line. npm plugin for Hardhat and a Foundry forge scripting plugin with full feature parity — no circuit expertise required.

Auditor Portal

Read-Only Registry Queries

Institutional auditors query the BSC mainnet registry by contract address via a Next.js portal. No wallet connection. No privileged access. Just cryptographic verification against live on-chain state.

Reliability

Production Proof Server

BullMQ + Redis job queue with retry logic. Sub-30-second end-to-end proof latency benchmarked on BEP-20 mint circuits. IPFS-pinned circuit artifacts via Pinata for long-term reproducibility.

Coverage

BEP-20 & BEP-177 Event Hooks

Full coverage for BEP-20 privileged events today. BEP-177 governance execution support including proposal ID binding and quorum threshold verification landing in Q2 2026.
